# Руководство пользователя pmetal-midi

## Для кого это руководство

Это руководство написано для музыканта, который создает power metal баллады в Suno
и хочет перевести их в качественный MIDI для виртуальных инструментов (Shreddage 3.5 Hydra, Darkwall и др.).
Никаких знаний о программировании, серверах или MCP не требуется.

---

## Часть 1. Словарь терминов

### Аудио и MIDI

| Термин | Объяснение |
|--------|------------|
| **WAV** | Аудиоформат без сжатия. Сохраняет живое звучание инструмента. Большой размер файла. |
| **Стемы (Stems)** | Отдельные аудиодорожки песни: вокал, гитара, бас, ударные. Suno отдает их как WAV-файлы. |
| **MIDI** | Цифровые ноты. Не содержит звука — только инструкции: "сыграй ноту До на 3-й октаве громкостью 80 длительностью четверть". |
| **Flat MIDI** | "Плоский" MIDI — все ноты имеют одинаковую громкость, без pitch bend, без динамики. Такой файл получается после транскрипции и "чистки" в Guitar Pro 8. Ноты правильные, но звучит как робот. |
| **Expressive MIDI** | "Экспрессивный" MIDI — результат транскрипции живого аудио (Neural Note или audio-to-midi). Содержит реальную динамику, pitch bend, микро-тайминг. Но ноты могут быть неточными. |
| **Hybrid MIDI** | Результат нашей обработки: правильные ноты из flat MIDI + живая динамика из expressive MIDI. Звучит как живой музыкант. |
| **Velocity** | Громкость/сила удара ноты в MIDI (от 1 до 127). Velocity 40 = тихо, 100 = громко, 127 = максимум. |
| **Pitch Bend** | Подтяжка струны. Плавное изменение высоты ноты вверх или вниз. В MIDI выражается числом от -8192 до +8191. |
| **Ticks per Beat (TPB)** | Разрешение MIDI-файла. Стандартно 480 тиков на четвертную ноту. Чем больше — тем точнее тайминг. |
| **Track / Дорожка** | Одна линия инструмента в MIDI-файле. Например: "Solo Guitar", "Rhythm Guitar L", "Electric Bass". |

### Обработка

| Термин | Объяснение |
|--------|------------|
| **Транскрипция** | Перевод аудио (WAV) в MIDI. Рекомендован ТОЛЬКО Neural Note (85-95% точность). Klang.io запрещён (60-70% с артефактами). |
| **Fuzzy Matching** | "Нечеткое сопоставление" — алгоритм, который находит пары нот между flat и expressive MIDI, допуская небольшие расхождения во времени и высоте. |
| **Match Rate** | Процент нот из flat MIDI, для которых найдена пара в expressive MIDI. 78% = 78 из 100 нот сопоставлены. |
| **Quantization** | Выравнивание нот по ритмической сетке (четверти, восьмые, шестнадцатые). |
| **Humanization** | Небольшое отклонение от идеальной сетки, чтобы звучало живее. |
| **Savitzky-Golay** | Математический фильтр для сглаживания pitch bend. Убирает резкие скачки, сохраняя общую форму кривой. |
| **Quality Score** | Оценка качества результата от 0.0 до 1.0. Выше 0.70 = хороший результат. |
| **Self-correction** | Автоматическая повторная обработка с подобранными параметрами, если первый результат не прошел проверку качества. |

### Инфраструктура

| Термин | Объяснение |
|--------|------------|
| **Alma** | Сервер (Brix мини-ПК), на котором работает система обработки. |
| **Контейнер (Container)** | Изолированная среда на сервере, в которой запущена программа pmetal-midi. Как виртуальная машина, но легче. |
| **MCP (Model Context Protocol)** | Протокол, позволяющий Claude Desktop вызывать внешние программы как "инструменты". Claude пишет команду → программа на сервере её выполняет → результат возвращается в чат. |
| **MCP Server** | Программа, которая принимает команды от Claude Desktop и выполняет их. В нашем случае — pmetal-midi на сервере alma. |

## Часть 2. Какую вкладку использовать в Claude Desktop

На скриншоте видны три вкладки: **Chat**, **Cowork**, **Code**.

### Используйте вкладку Chat

Вкладка **Chat** — это то, что вам нужно. Именно в ней Claude может вызывать
MCP-инструменты (обработку MIDI).

| Вкладка | Для чего | Подходит ли для pmetal-midi |
|---------|----------|-----------------------------|
| **Chat** | Разговор с Claude + вызов инструментов (MCP) | **Да, используйте это** |
| **Cowork** | Claude выполняет фоновые задачи в папках | Нет |
| **Code** | Автодополнение кода в редакторе | Нет |

### Какую модель выбрать

В правом нижнем углу окна Chat можно выбрать модель.

| Модель | Стоимость | Когда использовать |
|--------|-----------|--------------------|
| **Haiku 4.5** | Самая дешевая | Рутинные операции: загрузка файлов, запуск обработки, просмотр результатов |
| **Sonnet** | Средняя | Анализ результатов, подбор параметров, если Haiku не справляется |
| **Opus** | Дорогая | Не нужна для этой задачи |

**Рекомендация**: начинайте с **Haiku 4.5** для экономии. Она отлично справляется
с вызовом инструментов. Переключайтесь на Sonnet только если Haiku дает плохие советы
по параметрам обработки.

## Часть 3. Как увидеть, что интеграция работает

### Шаг 1. Перезапустите Claude Desktop

После добавления MCP-сервера нужно **полностью закрыть и открыть Claude Desktop**
(Cmd+Q, затем снова открыть). Просто закрыть окно недостаточно.

### Шаг 2. Откройте вкладку Chat

Переключитесь с Cowork на **Chat**.

### Шаг 3. Проверьте наличие инструментов

В окне чата, внизу рядом с полем ввода, должна появиться иконка
**молотка (🔨)** или надпись с количеством доступных инструментов (например, "12 tools").

Нажмите на неё — откроется список из 12 инструментов:
- `upload_file` — загрузка файла из чата на сервер
- `download_file` — скачивание файла с сервера в чат
- `eq_filter` — **EQ-фильтрация WAV** перед Neural Note (разделение соло/ритм гитары)
- `extract_track` — **извлечение одной дорожки** из мультидорожечного MIDI
- `merge_midi` — слияние MIDI файлов
- `analyze_quality` — анализ качества MIDI
- `analyze_audio` — анализ WAV-файла (темп, транзиенты, спектр)
- `run_workflow` — пакетная обработка всех пар файлов в директории
- `midi_info` — информация о MIDI-файле
- `list_files` — список файлов на сервере
- `get_status` — статус сервера (CPU, RAM, версия)
- `get_processing_log` — последние строки лога обработки

Если иконки нет — значит MCP не подключен. Проверьте:
1. Закрыли ли вы Claude Desktop полностью (**Cmd+Q**, не просто закрыли окно)?
2. Запущен ли контейнер: `make status` в папке проекта
3. Доступен ли HTTP MCP: `curl -s http://192.168.50.103:8200/mcp` (должен ответить)
4. Установлен ли Node.js: `npx --version` (нужен для mcp-remote)

Конфиг Claude Desktop находится по адресу:
`~/Library/Application Support/Claude/claude_desktop_config.json`

Он должен содержать:
```json
{
  "mcpServers": {
    "pmetal-midi": {
      "command": "npx",
      "args": ["-y", "mcp-remote", "http://192.168.50.103:8200/mcp", "--allow-http"]
    }
  }
}
```

Сервер MCP работает по HTTP (Streamable HTTP) на порту 8200 сервера alma.
Пакет `mcp-remote` (npm) служит мостом: Claude Desktop вызывает его как stdio-процесс,
а он подключается к HTTP MCP-серверу по сети. Никакого SSH для MCP не нужно.
Подключение надёжнее SSH: не отваливается от простоя, не требует keepalive.
IP `192.168.50.103` — это сервер alma в локальной сети.

Для автоматической настройки конфига можно выполнить:
```bash
make configure-claude
```

## Часть 4. Строгий рабочий процесс

### Схема (без вариаций)

```
1. Suno → Запросить стемы (ТОЛЬКО гитарный стем, НЕ полный MP3)
2. Гитарный стем → Neural Note (Monophonic, Electric Guitar, Medium)
3. Если соло+ритм в одном стеме → eq_filter для разделения ПЕРЕД Neural Note
4. Neural Note выход + Guitar Pro 8 ручная транскрипция → *_flat.mid + *_neural.mid
5. Если мультидорожечный MIDI → extract_track для извлечения целевой дорожки
6. merge_midi(flat, neural) со строгими параметрами
7. Результат → DAW + VST (Shreddage 3.5 Hydra / Darkwall)
```

### ЗАПРЕЩЕНО

| Что | Почему |
|-----|--------|
| Полный MP3 из Suno | Neural Note распознает удары барабанов как высоту тона |
| Klang.io | 60-70% точность с артефактами (ошибки октав, двойные ноты) |
| Стемы без Pro/Plus | Suno требует Pro/Plus для экспорта стемов |

### Рекомендуемые инструменты транскрипции

| Инструмент | Точность | Полифония | Бенды | Статус |
|------------|----------|-----------|-------|--------|
| Neural Note | 85-95% | Моно | Отличные | **РЕКОМЕНДОВАН** |
| Klang.io | 60-70% | Поли | Грубые | **ЗАПРЕЩЁН** |
| Melodyne | 90-98% | Моно/поли | Отличные | Альтернатива (если бюджет позволяет) |

### Шаг 1. Подготовьте файлы

Для каждой песни вам нужны два файла:

1. **Flat MIDI** (`song_flat.mid`):
   - Возьмите гитарный стем WAV из Suno (Pro/Plus)
   - Транскрибируйте через Neural Note (Monophonic, Electric Guitar, Medium)
   - Откройте результат в Guitar Pro 8
   - Вычистите ноты, исправьте ошибки
   - Экспортируйте как MIDI

2. **Expressive MIDI** (`song_neural.mid`):
   - Тот же WAV-стем → Neural Note
   - Сохраните MIDI-выход напрямую (без редактирования)

### Шаг 1.5. EQ-подготовка (если соло и ритм гитара в одном стеме)

Если в стеме смешаны соло и ритм гитара, пропустите WAV через EQ-фильтр
**ДО** транскрипции в Neural Note:

Загрузите WAV на сервер (через `make upload SRC=./guitar_stem.wav`), затем в чате:

> Примени фильтр solo_guitar к файлу guitar_stem.wav

Доступные пресеты:
| Пресет | Действие |
|--------|----------|
| `solo_guitar` | Highpass 800 Hz + boost 2-4 kHz (изоляция лид-гитары) |
| `rhythm_guitar` | Lowpass 2000 Hz + boost 200-500 Hz (изоляция ритм-гитары) |
| `bass` | Lowpass 800 Hz + boost 60-250 Hz (изоляция бас-гитары) |
| `custom` | Все параметры задаются вручную |

Скачайте отфильтрованный WAV → пропустите через Neural Note.

### Шаг 2. Загрузите файлы

**ВАЖНО**: Claude Desktop НЕ умеет передавать бинарные файлы (.mid, .wav) через чат.
НЕ прикрепляйте файлы к сообщениям — это не работает.

Используйте **веб-интерфейс** для загрузки:

1. Откройте в браузере: **http://192.168.50.103:8100**
2. Перетащите файл(ы) в окно браузера (или нажмите "Drop files here")
3. Нажмите **Upload**
4. Файлы появятся в списке ниже

Поддерживаются: `.mid`, `.midi`, `.wav`, `.flac`, `.mp3` (до 200 МБ)

Через тот же веб-интерфейс можно скачать результаты (кнопка ↓ рядом с файлом).

Альтернативно через терминал:
```bash
make upload SRC=./guitar_stem.wav
```

После загрузки, в чате Claude Desktop:
> Покажи файлы на сервере

### Шаг 2.5. Мультидорожечный MIDI

Если flat MIDI содержит несколько дорожек (мультитрек из Guitar Pro):

1. Вызовите `midi_info` чтобы увидеть все дорожки:
   > Покажи информацию о song_flat.mid

2. Извлеките нужную дорожку:
   > Извлеки дорожку 3 из song_flat.mid

3. Используйте извлечённый файл для merge.

### Шаг 3. Запустите обработку

> Объедини файлы song_flat.mid и song_neural.mid

Для мультитрек-файла указывайте конкретную дорожку:
> Объедини song_flat.mid и song_neural.mid, обработай только дорожку 3

### Шаг 4. Оцените результат

> Проанализируй качество выходного файла

Claude покажет детальный отчет:
```
  [+] density: 0.85
  [+] pitch_bend_continuity: 0.91
  [+] velocity_range: 0.78
  [+] timing_consistency: 0.82
  [-] match_rate: 0.62

  Overall: 0.79 — PASS
```

Если результат < 0.70, Claude укажет **конкретные значения** для повтора (не "попробуй"):
```
  match_rate < 0.5 → matching_window_ticks=200, pitch_tolerance=5
  velocity_range < 0.6 → velocity_boost=1.4
  pitch_bend_continuity < 0.6 → smoothing_window=15
```

### Шаг 5. Повторите с указанными параметрами

> Повтори обработку с matching_window_ticks=200 и velocity_boost=1.4

### Шаг 6. Скачайте результат

> Скачай результат обработки

Claude вызовет `download_file` и отдаст файл `song_hybrid.mid` в чат.

## Часть 6. Что делать, если что-то не работает

### Claude не видит инструменты (нет иконки молотка)

1. Полностью закройте Claude Desktop: **Cmd+Q**
2. Откройте терминал:
   ```bash
   cd ~/cursor-eval/pmetal-midi
   make status
   ```
3. Если контейнер не запущен:
   ```bash
   make start
   ```
4. Проверьте SSH:
   ```bash
   ssh alma "echo OK"
   ```
5. Проверьте MCP:
   ```bash
   make test-mcp
   ```
6. Запустите Claude Desktop заново

### Обработка завершилась ошибкой

Спросите Claude: "Что пошло не так?" — он покажет лог ошибки.

Частые причины:
- Файл не найден → проверьте `make ls-data`
- Пустые дорожки → проверьте midi_info
- Нет совпадений → файлы от разных песен или разных секций

### Низкий match rate (<30%)

- Убедитесь, что flat и expressive MIDI от одной песни
- Expressive MIDI должен быть транскрипцией той же гитарной партии
- Попробуйте увеличить matching_window_ticks до 200–240

### Контейнер упал

```bash
make logs     # посмотреть что случилось
make restart  # перезапустить
```

## Часть 7. Возможности системы (подробнее)

### Автоматический самокорректирующий цикл

При обработке система автоматически:
1. **Анализирует качество** по 5 метрикам (плотность нот, pitch bend, velocity, тайминг, match rate)
2. Если оценка < 0.70, **автоматически корректирует параметры** (увеличивает окно матчинга, boost velocity и т.д.)
3. Повторяет обработку до 3 раз с новыми параметрами
4. **FeedbackLoop** отслеживает прогресс: если оценка ухудшается — остановка

### Пакетная обработка (run_workflow)

Если у вас несколько песен, положите их все в `/data/input/` с суффиксами:
- `song1_flat.mid` + `song1_neural.mid`
- `song2_flat.mid` + `song2_neural.mid`

Скажите Claude: "Обработай все файлы в /data/input" — система найдёт все пары и обработает каждую.

### EQ-фильтрация (eq_filter)

Если соло и ритм гитара объединены в одном стеме из Suno, прогоните WAV
через EQ-фильтр **перед** транскрипцией в Neural Note:

- **solo_guitar**: highpass 800 Hz + boost 2-4 kHz → изолирует лид-гитару
- **rhythm_guitar**: lowpass 2000 Hz + boost 200-500 Hz → изолирует ритм
- **bass**: lowpass 800 Hz + boost 60-250 Hz → изолирует бас
- **custom**: все параметры задаются вручную

Это повышает точность Neural Note при распознавании.

### Работа с мультидорожечным MIDI (extract_track)

Если пользователь предоставляет мультитрек-MIDI из Guitar Pro (несколько дорожек):
1. `midi_info` показывает все дорожки с индексами
2. `extract_track` извлекает одну дорожку в отдельный файл
3. `merge_midi` с `target_tracks` обрабатывает только указанные дорожки

Каждый раз обрабатывается **одна дорожка за один проход**.

### Анализ аудио

Если есть WAV-стем гитары от Suno, загрузите на сервер (`make upload SRC=./guitar.wav`)
и скажите в чате: "Проанализируй аудио guitar.wav"

Система определит:
- **Темп** (BPM) — автоматически или вручную
- **Транзиенты** (атаки нот) — для корректировки velocity
- **Спектральные пики** — для валидации pitch bend
- **Биты** — для проверки тайминга

### Безопасность и мониторинг

- Все пути файлов валидируются: доступ только к `/data/*`
- Rate limiting: максимум 30 вызовов в минуту
- Мониторинг ресурсов: CPU и RAM отслеживаются через `get_status`

## Часть 8. Администрирование (для продвинутых)

### Полезные make-команды

```bash
make help          # список всех команд
make status        # статус контейнера
make logs          # логи в реальном времени
make shell         # зайти внутрь контейнера
make ls-data       # список всех файлов данных
make redeploy      # обновить код и перезапустить

```

### Обновление после изменений в коде

```bash
make redeploy
```

Это выполнит sync → rebuild → restart. Данные в `/data/` сохраняются.

### Ручной запуск из командной строки (внутри контейнера)

```bash
make shell
# Внутри контейнера:
pmetal info /data/input/song_flat.mid
pmetal merge /data/input/flat.mid /data/input/neural.mid -o /data/output/ -v
pmetal analyze /data/output/song_hybrid.mid
pmetal extract /data/input/guitar.wav
pmetal config show
pmetal config validate /data/config/custom.yaml
```

### Доступные CLI-команды

| Команда | Описание |
|---------|----------|
| `pmetal merge` | Слияние flat + expressive MIDI с Rich-прогрессом |
| `pmetal analyze` | Анализ качества MIDI (таблица метрик) |
| `pmetal extract` | Анализ аудио (темп, транзиенты, спектр) |
| `pmetal info` | Информация о MIDI-файле (дорожки, ноты) |
| `pmetal config show` | Показать текущую конфигурацию |
| `pmetal config validate` | Проверить валидность YAML-конфигурации |
